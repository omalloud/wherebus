# WhereBus 🚌

A real-time bus tracking web application that lets passengers locate buses on a live map, view estimated arrival times, and plan their journeys — while giving bus operators a dedicated dashboard to broadcast their location.

## Features

- **Live Bus Map** — Track active buses in real time on an interactive map powered by Amazon Location Service.
- **Route Estimation** — Calculate the estimated travel time between a passenger's pickup point and a selected bus line.
- **Passenger Interface** — Search for a station and a bus line; the map flies to the bus and draws the route.
- **Driver Interface** — Dedicated page for drivers to register and share their real-time GPS position.
- **Company Registration** — Transport companies can register their fleet and manage bus lines.
- **Authentication** — Secure login and sign-up flows backed by AWS Amplify Auth.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 18, React Router v6, SASS |
| Map | Amazon Location Service, AWS Amplify MapView, react-map-gl |
| Real-time data | Firebase Realtime Database |
| Auth & backend | AWS Amplify |
| Routing API | AWS Location Service — Route Calculator |
| Deployment | Firebase Hosting / GitHub Pages |

## Prerequisites

- [Node.js](https://nodejs.org/) v16 or later and npm
- An **AWS account** with Amplify configured (`amplify/` directory included)
- A **Firebase project** with Realtime Database enabled
- Firebase credentials file (`src/firebase-config.js` or equivalent) — see [Firebase setup](https://firebase.google.com/docs/web/setup)
- AWS Amplify exports file (`src/aws-exports.js`) — generated by `amplify push`

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/omalloud/wherebus.git
cd wherebus
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment

Make sure the following auto-generated config files are present (they are excluded from version control):

- `src/aws-exports.js` — generated by the Amplify CLI after `amplify push`
- Firebase config — initialised in `src/FirebaseFunction.js`

If setting up from scratch, run:

```bash
amplify init
amplify push
```

### 4. Start the development server

```bash
npm start
```

Open [http://localhost:3000](http://localhost:3000) in your browser. The app hot-reloads on file changes.

## Available Scripts

| Command | Description |
|---|---|
| `npm start` | Start the development server on port 3000 |
| `npm test` | Run the test suite in interactive watch mode |
| `npm run build` | Create an optimised production build in the `build/` folder |
| `npm run deploy` | Build and publish to GitHub Pages |

## Deployment

### Firebase Hosting

```bash
npm run build
firebase deploy
```

### GitHub Pages

```bash
npm run deploy
```

> The `predeploy` script runs `npm run build` automatically before publishing.

## Project Structure

```

├── App.js               # Root component and routing
└── index.js             # Application entry point
amplify/                 # AWS Amplify backend configuration
public/                  # Static assets
```
